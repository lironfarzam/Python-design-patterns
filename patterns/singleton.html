Singleton Pattern

חזרה לדף הבית

מטרת התבנית

תבנית Singleton מבטיחה שקיים רק מופע אחד של מחלקה, ומספקת נקודת גישה גלובלית אליו.

מתי להשתמש?

  • כאשר רוצים לוודא שקיים רק מופע אחד של מחלקה
  • כאשר רוצים לספק נקודת גישה גלובלית
  • כאשר רוצים לשלוט בגישה למשאב משותף
  • כאשר רוצים לנהל מצב גלובלי

דוגמה בסיסית 

class Singleton:
    _instance = None

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

    def __init__(self):
        if not hasattr(self, 'initialized'):
            self.initialized = True
            self.value = "ערך ראשוני"

# שימוש
singleton1 = Singleton()
singleton2 = Singleton()

print(singleton1 is singleton2)  # True
print(singleton1.value)          # ערך ראשוני
print(singleton2.value)          # ערך ראשוני

singleton1.value = "ערך חדש"
print(singleton2.value)          # ערך חדש

יתרונות

  • מבטיח שקיים רק מופע אחד
  • מספק נקודת גישה גלובלית
  • מאפשר שליטה בגישה למשאבים
  • מאפשר ניהול מצב גלובלי

חסרונות

  • עלול להקשות על בדיקות
  • עלול ליצור תלות חזקה
  • עלול להפר את עקרון היחידות
  • עלול ליצור קוד קשה לתחזוקה

השוואה לתבניות אחרות

תבנית הבדל
Factory Method מתמקד ביצירת אובייקטים, בעוד Singleton מתמקד במופע יחיד
Prototype מתמקד בהעתקת אובייקטים, בעוד Singleton מתמקד במופע יחיד
