Adapter Pattern

חזרה לדף הבית

מטרת התבנית

תבנית Adapter מאפשרת לממשקים לא תואמים לעבוד יחד על ידי המרת הממשק של מחלקה אחת לממשק שהלקוח מצפה לו.

מתי להשתמש?

  • כאשר רוצים להשתמש במחלקה קיימת אך הממשק שלה לא תואם לצרכים
  • כאשר רוצים ליצור מחלקה שניתן לעבוד איתה עם מחלקות אחרות
  • כאשר רוצים לספק ממשק אחיד למספר ממשקים קיימים
  • כאשר רוצים להסתיר את המימוש של מחלקה קיימת

דוגמה בסיסית 

class Target:
    def request(self) -> str:
        return "תוצאה של Target"

class Adaptee:
    def specific_request(self) -> str:
        return "תוצאה של Adaptee"

class Adapter(Target):
    def __init__(self, adaptee: Adaptee):
        self.adaptee = adaptee

    def request(self) -> str:
        return f"Adapter: {self.adaptee.specific_request()}"

# שימוש
def client_code(target: Target) -> None:
    print(target.request())

if __name__ == "__main__":
    target = Target()
    client_code(target)

    adaptee = Adaptee()
    adapter = Adapter(adaptee)
    client_code(adapter)

יתרונות

  • מאפשרת שימוש במחלקות קיימות
  • מאפשרת שימוש במחלקות ממקורות שונים
  • מאפשרת שימוש במחלקות עם ממשקים שונים
  • מאפשרת שימוש במחלקות ללא שינוי הקוד שלהן

חסרונות

  • עלולה ליצור קוד מורכב
  • עלולה ליצור קוד קשה לתחזוקה
  • עלולה ליצור קוד קשה להבנה
  • עלולה ליצור קוד קשה לבדיקה

השוואה לתבניות אחרות

תבנית הבדל
Bridge מפרידה בין ממשק למימוש, בעוד Adapter מתמקדת בהתאמת ממשקים
Decorator מוסיפה תפקוד לאובייקט, בעוד Adapter מתמקדת בהתאמת ממשקים

תרגול

נסה ליישם את תבנית Adapter במקרים הבאים:

  1. התאמת ממשק של ספרייה חיצונית
  2. התאמת ממשק של מערכת ישנה
  3. התאמת ממשק של שירות חיצוני
