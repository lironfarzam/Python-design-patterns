Adapter Pattern

חזרה לדף הבית

מהי תבנית Adapter?

תבנית Adapter היא תבנית עיצוב מבנית המאפשרת לממשקים לא תואמים לעבוד יחד. התבנית משמשת כאשר יש צורך להשתמש במחלקה קיימת, אך הממשק שלה אינו תואם את הממשק הנדרש.

הסבר למתחילים:

דמיינו שיש לכם מטען טלפון עם חיבור מיוחד, אבל השקע בקיר הוא מסוג אחר. מתאם (Adapter) מאפשר לחבר את המטען לשקע על ידי המרת החיבור. תבנית Adapter עובדת באותו אופן - היא מאפשרת לממשקים שונים לעבוד יחד.

הסבר למתקדמים:

התבנית מיישמת את עקרון ה-Adapter Pattern על ידי שימוש במחלקת מתאם (Adapter) שממירה את הממשק של מחלקה קיימת לממשק הנדרש. זה מאפשר שימוש חוזר בקוד קיים ומאפשר אינטגרציה בין מערכות שונות.

מתי להשתמש בתבנית?

למתחילים:

  • כאשר אתם צריכים להשתמש במחלקה קיימת עם ממשק שונה
  • כאשר אתם צריכים ליצור מחלקה שתתמוך בממשקים שונים
  • כאשר אתם צריכים לאפשר אינטגרציה בין מערכות שונות

למתקדמים:

  • כאשר יש צורך להשתמש במחלקה קיימת עם ממשק שונה
  • כאשר יש צורך ליצור מחלקה שתתמוך בממשקים שונים
  • כאשר יש צורך לאפשר אינטגרציה בין מערכות שונות
  • כאשר יש צורך לשמור על עקרון ה-Open/Closed Principle
  • כאשר יש צורך לאפשר שימוש חוזר בקוד קיים
  • כאשר יש צורך לאפשר שינוי ממשקים ללא שינוי הקוד הקיים

מבנה התבנית

הסבר בסיסי:

התבנית מורכבת משלושה חלקים עיקריים:

  • Target - הממשק שהלקוח מצפה לו
  • Adaptee - המחלקה הקיימת שצריך להתאים
  • Adapter - המחלקה שמתאימה את Adaptee ל-Target

הסבר מתקדם:

התבנית מורכבת מהמרכיבים הבאים:

  • Target - הממשק שהלקוח מצפה לו
  • Adaptee - המחלקה הקיימת שצריך להתאים
  • Adapter - המחלקה שמתאימה את Adaptee ל-Target
  • Client - המשתמש ב-Target דרך Adapter

היחסים בין המרכיבים:

  • Client משתמש ב-Target
  • Adapter מיישם את Target
  • Adapter משתמש ב-Adaptee
  • Adaptee מספק את הפונקציונליות הבסיסית

דוגמה בסיסית

הסבר למתחילים:

בדוגמה הבאה, אנחנו יוצרים מערכת לניהול מסמכים. יש לנו מחלקה קיימת שמטפלת במסמכי PDF, אבל אנחנו רוצים להשתמש בה עם ממשק אחיד לכל סוגי המסמכים.

הסבר למתקדמים:

הדוגמה מדגימה את השימוש ב-Adapter Pattern ליצירת ממשק אחיד למסמכים. הממשק המופשט מגדיר את הפעולות הבסיסיות, והמתאם מספק את המימוש הספציפי.

from abc import ABC, abstractmethod

class Document(ABC):
    @abstractmethod
    def open(self):
        pass
    
    @abstractmethod
    def save(self):
        pass

class PDFDocument:
    def open_pdf(self):
        print("Opening PDF document")
    
    def save_pdf(self):
        print("Saving PDF document")

class PDFAdapter(Document):
    def __init__(self, pdf_document: PDFDocument):
        self.pdf_document = pdf_document
    
    def open(self):
        self.pdf_document.open_pdf()
    
    def save(self):
        self.pdf_document.save_pdf()

# שימוש
if __name__ == "__main__":
    # יצירת מסמך PDF
    pdf = PDFDocument()
    
    # יצירת מתאם
    adapter = PDFAdapter(pdf)
    
    # שימוש במתאם
    adapter.open()
    adapter.save()

דוגמה מעשית - מערכת תשלומים

הסבר למתחילים:

בדוגמה זו, אנחנו יוצרים מערכת תשלומים שתומכת בשיטות תשלום שונות. יש לנו מחלקות קיימות שמטפלות בכל שיטת תשלום, אבל אנחנו רוצים להשתמש בהן עם ממשק אחיד.

הסבר למתקדמים:

הדוגמה מדגימה את השימוש ב-Adapter Pattern ליצירת ממשק אחיד לתשלומים. הממשק המופשט מגדיר את הפעולות הבסיסיות, והמתאמים מספקים את המימוש הספציפי לכל שיטת תשלום.

from abc import ABC, abstractmethod
from typing import Dict, Any

class PaymentProcessor(ABC):
    @abstractmethod
    def process_payment(self, amount: float, details: Dict[str, Any]) -> bool:
        pass

class CreditCardProcessor:
    def charge_credit_card(self, amount: float, card_number: str, expiry_date: str, cvv: str) -> bool:
        print(f"Charging credit card {card_number} for ${amount}")
        return True

class PayPalProcessor:
    def make_payment(self, amount: float, email: str, password: str) -> bool:
        print(f"Making PayPal payment from {email} for ${amount}")
        return True

class CreditCardAdapter(PaymentProcessor):
    def __init__(self, processor: CreditCardProcessor):
        self.processor = processor
    
    def process_payment(self, amount: float, details: Dict[str, Any]) -> bool:
        return self.processor.charge_credit_card(
            amount,
            details["card_number"],
            details["expiry_date"],
            details["cvv"]
        )

class PayPalAdapter(PaymentProcessor):
    def __init__(self, processor: PayPalProcessor):
        self.processor = processor
    
    def process_payment(self, amount: float, details: Dict[str, Any]) -> bool:
        return self.processor.make_payment(
            amount,
            details["email"],
            details["password"]
        )

# שימוש
if __name__ == "__main__":
    # יצירת מעבדי תשלום
    credit_card_processor = CreditCardProcessor()
    paypal_processor = PayPalProcessor()
    
    # יצירת מתאמים
    credit_card_adapter = CreditCardAdapter(credit_card_processor)
    paypal_adapter = PayPalAdapter(paypal_processor)
    
    # שימוש במתאמים
    credit_card_details = {
        "card_number": "1234-5678-9012-3456",
        "expiry_date": "12/25",
        "cvv": "123"
    }
    credit_card_adapter.process_payment(100.0, credit_card_details)
    
    paypal_details = {
        "email": "user@example.com",
        "password": "password123"
    }
    paypal_adapter.process_payment(50.0, paypal_details)

השוואה לתבניות אחרות

למתחילים:

תבנית הבדל
Bridge מתמקדת בהפרדה בין ממשק למימוש
Decorator מתמקדת בהוספת תכונות לאובייקט

למתקדמים:

תבנית הבדל
Bridge מתמקדת בהפרדה בין ממשק למימוש, בעוד Adapter מתמקדת בהתאמת ממשקים
Decorator מתמקדת בהוספת תכונות לאובייקט, בעוד Adapter מתמקדת בהתאמת ממשקים
Facade מתמקדת בספק ממשק פשוט למערכת מורכבת, בעוד Adapter מתמקדת בהתאמת ממשקים
Proxy מתמקדת בשליטה בגישה לאובייקט, בעוד Adapter מתמקדת בהתאמת ממשקים

תרגול

למתחילים:

נסה ליישם את תבנית Adapter במקרים הבאים:

  1. מערכת ניהול מסמכים עם תמיכה בפורמטים שונים
  2. מערכת תשלומים עם תמיכה בשיטות תשלום שונות
  3. מערכת תקשורת עם תמיכה בפרוטוקולים שונים

למתקדמים:

נסה ליישם את תבנית Adapter במקרים הבאים:

  1. מערכת אחסון קבצים עם תמיכה בשירותי אחסון שונים
  2. מערכת דיווח עם תמיכה בפורמטים שונים
  3. מערכת תקשורת עם תמיכה בפרוטוקולים שונים
  4. מערכת עיבוד תמונות עם תמיכה בפורמטים שונים
