Factory Method Pattern

חזרה לדף הבית

מטרת התבנית

תבנית Factory Method מגדירה ממשק ליצירת אובייקט, אך מאפשרת למחלקות היורשות להחליט איזה מחלקה ליצור.

מתי להשתמש?

  • כאשר המחלקה לא יודעת מראש איזה סוג של אובייקטים היא צריכה ליצור
  • כאשר המחלקה רוצה שתחלקות היורשות שלה יקבעו איזה אובייקט ליצור
  • כאשר רוצים להפריד בין יצירת האובייקט לשימוש בו

דוגמה בסיסית 

from abc import ABC, abstractmethod

# ממשק המוצר
class Document(ABC):
    @abstractmethod
    def create_page(self):
        pass

# מימושים קונקרטיים
class Resume(Document):
    def create_page(self):
        return "יצירת דף קורות חיים"

class Report(Document):
    def create_page(self):
        return "יצירת דף דוח"

# ממשק היוצר
class DocumentCreator(ABC):
    @abstractmethod
    def create_document(self) -> Document:
        pass

    def render_document(self):
        document = self.create_document()
        return document.create_page()

# יוצרים קונקרטיים
class ResumeCreator(DocumentCreator):
    def create_document(self) -> Document:
        return Resume()

class ReportCreator(DocumentCreator):
    def create_document(self) -> Document:
        return Report()

# שימוש
def create_document(creator: DocumentCreator):
    return creator.render_document()

# יצירת מסמך לפי סוג
resume = create_document(ResumeCreator())
report = create_document(ReportCreator())

print(resume)  # יצירת דף קורות חיים
print(report)  # יצירת דף דוח

יתרונות

  • מאפשר הרחבה קלה של סוגי המוצרים
  • מפריד בין יצירת האובייקט לשימוש בו
  • מאפשר למחלקות היורשות להחליט איזה אובייקט ליצור
  • מקדם עקרון היחידות (Single Responsibility)

חסרונות

  • מגדיל את מספר המחלקות בקוד
  • עלול להיות מורכב מדי עבור מקרים פשוטים
  • דורש יצירת מחלקה נפרדת לכל סוג מוצר

השוואה לתבניות אחרות

תבנית הבדל
Abstract Factory מתמקד ביצירת משפחות של אובייקטים, בעוד Factory Method מתמקד ביצירת אובייקט בודד
Builder מתמקד בבניית אובייקט מורכב צעד אחר צעד, בעוד Factory Method מתמקד ביצירת אובייקט בודד

תרגול

נסה ליישם את תבנית Factory Method במקרים הבאים:

  1. יצירת סוגים שונים של כלי רכב (מכונית, אופנוע, משאית)
  2. יצירת סוגים שונים של מסמכים (PDF, Word, Excel)
  3. יצירת סוגים שונים של משתמשים (מנהל, עובד, לקוח)
