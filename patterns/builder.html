Builder Pattern

חזרה לדף הבית

מטרת התבנית

תבנית Builder מאפשרת בנייה של אובייקטים מורכבים בשלבים, ומפרידה בין תהליך הבנייה לייצוג הסופי של האובייקט.

מתי להשתמש?

  • כאשר האובייקט שיש לבנות הוא מורכב ודורש מספר שלבים
  • כאשר רוצים ליצור גרסאות שונות של אותו אובייקט
  • כאשר רוצים להפריד בין תהליך הבנייה לייצוג הסופי
  • כאשר רוצים לספק API נוח לבניית אובייקטים מורכבים

דוגמה בסיסית 

from abc import ABC, abstractmethod
from typing import Any

class Product:
    def __init__(self) -> None:
        self.parts: list[str] = []

    def add_part(self, part: str) -> None:
        self.parts.append(part)

    def show_parts(self) -> None:
        print("חלקי המוצר:", ", ".join(self.parts))

class Builder(ABC):
    @abstractmethod
    def build_part_a(self) -> None:
        pass

    @abstractmethod
    def build_part_b(self) -> None:
        pass

    @abstractmethod
    def get_result(self) -> Product:
        pass

class ConcreteBuilder(Builder):
    def __init__(self) -> None:
        self._product = Product()

    def build_part_a(self) -> None:
        self._product.add_part("חלק A")

    def build_part_b(self) -> None:
        self._product.add_part("חלק B")

    def get_result(self) -> Product:
        return self._product

class Director:
    def __init__(self, builder: Builder) -> None:
        self._builder = builder

    def construct(self) -> Product:
        self._builder.build_part_a()
        self._builder.build_part_b()
        return self._builder.get_result()

# שימוש
def client_code() -> None:
    builder = ConcreteBuilder()
    director = Director(builder)
    product = director.construct()
    product.show_parts()

# דוגמה לשימוש
if __name__ == "__main__":
    client_code()

יתרונות

  • מאפשרת בנייה של אובייקטים מורכבים בשלבים
  • מפרידה בין תהליך הבנייה לייצוג הסופי
  • מאפשרת יצירת גרסאות שונות של אותו אובייקט
  • מספקת API נוח לבניית אובייקטים מורכבים

חסרונות

  • עלולה להקשות על הבנת הקוד
  • עלולה ליצור מספר רב של מחלקות
  • עלולה להאט את הביצועים

השוואה לתבניות אחרות

תבנית הבדל
Abstract Factory מתמקדת ביצירת משפחות של אובייקטים, בעוד Builder מתמקדת בבניית אובייקט מורכב
Factory Method מתמקדת ביצירת אובייקט בודד, בעוד Builder מתמקדת בבניית אובייקט מורכב

  3. מערכת בניית תפריטים
