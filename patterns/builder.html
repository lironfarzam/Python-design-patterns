Builder Pattern

חזרה לדף הבית

מהי תבנית Builder?

תבנית Builder היא תבנית עיצוב יצירתית המפרידה בין בניית אובייקט מורכב לייצוג שלו, כך שאותו תהליך בנייה יכול ליצור ייצוגים שונים. התבנית מאפשרת בנייה הדרגתית של אובייקט מורכב.

הסבר למתחילים:

דמיינו שאתם בונים בית. במקום לנסות לבנות את כל הבית בבת אחת, אתם בונים אותו שלב אחר שלב: קודם יסודות, אחר כך קירות, גג, חלונות וכו'. תבנית Builder עובדת באותו אופן - היא מאפשרת לבנות אובייקט מורכב שלב אחר שלב.

הסבר למתקדמים:

התבנית מיישמת את עקרון ה-Single Responsibility Principle (SRP) על ידי הפרדה בין הלוגיקה של בניית האובייקט לייצוג שלו. זה מאפשר יצירת אובייקטים מורכבים בצורה מודולרית וגמישה, תוך שמירה על קוד נקי וקריא.

מתי להשתמש בתבנית?

למתחילים:

  • כאשר אתם צריכים ליצור אובייקט מורכב עם הרבה חלקים
  • כאשר אתם רוצים לבנות את האובייקט שלב אחר שלב
  • כאשר אתם רוצים ליצור גרסאות שונות של אותו אובייקט

למתקדמים:

  • כאשר האובייקט המורכב דורש תהליך בנייה מורכב
  • כאשר יש צורך ליצור גרסאות שונות של אותו אובייקט
  • כאשר רוצים להפריד בין הלוגיקה של בניית האובייקט לייצוג שלו
  • כאשר רוצים לאפשר בנייה הדרגתית של האובייקט
  • כאשר רוצים להסתיר את המורכבות של בניית האובייקט
  • כאשר רוצים לאפשר שימוש חוזר בלוגיקת הבנייה
  • כאשר רוצים לאפשר הרחבה קלה של תהליך הבנייה

מבנה התבנית

הסבר בסיסי:

התבנית מורכבת משלושה חלקים עיקריים:

  • Director - מנהל את תהליך הבנייה
  • Builder - מגדיר את הממשק לבניית האובייקט
  • Product - האובייקט המורכב שנבנה

הסבר מתקדם:

התבנית מורכבת מהמרכיבים הבאים:

  • Director - מנהל את תהליך הבנייה ומשתמש ב-Builder
  • Builder - ממשק מופשט המגדיר את הפעולות לבניית האובייקט
  • ConcreteBuilder - מימוש של Builder, מיישם את הפעולות לבניית האובייקט
  • Product - האובייקט המורכב שנבנה

היחסים בין המרכיבים:

  • Director משתמש ב-Builder לבניית האובייקט
  • ConcreteBuilder מיישם את Builder ויוצר את האובייקט
  • Product הוא האובייקט המורכב שנבנה

דוגמה בסיסית

הסבר למתחילים:

בדוגמה הבאה, אנחנו יוצרים מערכת לבניית בית. המחלקה הבסיסית מגדירה איך לבנות בית, והמחלקות היורשות מיישמות את הבנייה עבור כל סוג בית.

הסבר למתקדמים:

הדוגמה מדגימה את השימוש ב-Builder Pattern לבניית אובייקט מורכב. הממשק המופשט מגדיר את הפעולות לבניית האובייקט, והמימושים הקונקרטיים מספקים את המימוש הספציפי.

from abc import ABC, abstractmethod

class House:
    def __init__(self):
        self.walls = []
        self.doors = []
        self.windows = []
        self.roof = None
    
    def __str__(self):
        return f"House with {len(self.walls)} walls, {len(self.doors)} doors, {len(self.windows)} windows, and {self.roof}"

class HouseBuilder(ABC):
    def __init__(self):
        self.house = House()
    
    @abstractmethod
    def build_walls(self):
        pass
    
    @abstractmethod
    def build_doors(self):
        pass
    
    @abstractmethod
    def build_windows(self):
        pass
    
    @abstractmethod
    def build_roof(self):
        pass
    
    def get_house(self):
        return self.house

class WoodenHouseBuilder(HouseBuilder):
    def build_walls(self):
        self.house.walls = ["Wooden walls"]
    
    def build_doors(self):
        self.house.doors = ["Wooden doors"]
    
    def build_windows(self):
        self.house.windows = ["Wooden windows"]
    
    def build_roof(self):
        self.house.roof = "Wooden roof"

class BrickHouseBuilder(HouseBuilder):
    def build_walls(self):
        self.house.walls = ["Brick walls"]
    
    def build_doors(self):
        self.house.doors = ["Brick doors"]
    
    def build_windows(self):
        self.house.windows = ["Brick windows"]
    
    def build_roof(self):
        self.house.roof = "Brick roof"

class HouseDirector:
    def __init__(self, builder: HouseBuilder):
        self.builder = builder
    
    def build_house(self):
        self.builder.build_walls()
        self.builder.build_doors()
        self.builder.build_windows()
        self.builder.build_roof()
        return self.builder.get_house()

# שימוש
if __name__ == "__main__":
    # בניית בית מעץ
    wooden_builder = WoodenHouseBuilder()
    director = HouseDirector(wooden_builder)
    wooden_house = director.build_house()
    print(wooden_house)
    
    # בניית בית מלבנים
    brick_builder = BrickHouseBuilder()
    director = HouseDirector(brick_builder)
    brick_house = director.build_house()
    print(brick_house)

דוגמה מעשית - מערכת ניהול מסמכים

הסבר למתחילים:

בדוגמה זו, אנחנו יוצרים מערכת ניהול מסמכים שתומכת בסוגים שונים של מסמכים. המחלקה הבסיסית מגדירה איך לבנות מסמך, והמחלקות היורשות מיישמות את הבנייה עבור כל סוג מסמך.

הסבר למתקדמים:

הדוגמה מדגימה את השימוש ב-Builder Pattern לבניית אובייקט מורכב. הממשק המופשט מגדיר את הפעולות לבניית האובייקט, והמימושים הקונקרטיים מספקים את המימוש הספציפי.

from abc import ABC, abstractmethod
from datetime import datetime

class Document:
    def __init__(self):
        self.title = None
        self.author = None
        self.content = []
        self.created_at = None
        self.modified_at = None
    
    def __str__(self):
        return f"Document: {self.title}\nAuthor: {self.author}\nContent: {self.content}\nCreated: {self.created_at}\nModified: {self.modified_at}"

class DocumentBuilder(ABC):
    def __init__(self):
        self.document = Document()
    
    @abstractmethod
    def set_title(self, title: str):
        pass
    
    @abstractmethod
    def set_author(self, author: str):
        pass
    
    @abstractmethod
    def add_content(self, content: str):
        pass
    
    @abstractmethod
    def set_created_at(self):
        pass
    
    @abstractmethod
    def set_modified_at(self):
        pass
    
    def get_document(self):
        return self.document

class PDFDocumentBuilder(DocumentBuilder):
    def set_title(self, title: str):
        self.document.title = f"PDF: {title}"
    
    def set_author(self, author: str):
        self.document.author = author
    
    def add_content(self, content: str):
        self.document.content.append(f"PDF Content: {content}")
    
    def set_created_at(self):
        self.document.created_at = datetime.now()
    
    def set_modified_at(self):
        self.document.modified_at = datetime.now()

class HTMLDocumentBuilder(DocumentBuilder):
    def set_title(self, title: str):
        self.document.title = f"HTML: {title}"
    
    def set_author(self, author: str):
        self.document.author = author
    
    def add_content(self, content: str):
        self.document.content.append(f"
{content}
")
    
    def set_created_at(self):
        self.document.created_at = datetime.now()
    
    def set_modified_at(self):
        self.document.modified_at = datetime.now()

class DocumentDirector:
    def __init__(self, builder: DocumentBuilder):
        self.builder = builder
    
    def build_document(self, title: str, author: str, content: list):
        self.builder.set_title(title)
        self.builder.set_author(author)
        self.builder.set_created_at()
        
        for item in content:
            self.builder.add_content(item)
        
        self.builder.set_modified_at()
        return self.builder.get_document()

# שימוש
if __name__ == "__main__":
    # בניית מסמך PDF
    pdf_builder = PDFDocumentBuilder()
    director = DocumentDirector(pdf_builder)
    pdf_document = director.build_document(
        "Example PDF",
        "John Doe",
        ["First paragraph", "Second paragraph", "Third paragraph"]
    )
    print(pdf_document)
    
    # בניית מסמך HTML
    html_builder = HTMLDocumentBuilder()
    director = DocumentDirector(html_builder)
    html_document = director.build_document(
        "Example HTML",
        "Jane Doe",
        ["First paragraph", "Second paragraph", "Third paragraph"]
    )
    print(html_document)

יתרונות

למתחילים:

  • מאפשרת בנייה הדרגתית של אובייקט מורכב
  • מקלה על יצירת גרסאות שונות של אותו אובייקט
  • מפרידה בין הלוגיקה של בניית האובייקט לייצוג שלו

למתקדמים:

  • מאפשרת הרחבה של תהליך הבנייה
  • מספקת ממשק אחיד לבניית האובייקט
  • מאפשרת החלפה דינמית של תהליך הבנייה
  • מקלה על בדיקות (Testing) של המערכת
  • מאפשרת שימוש ב-Mock Objects לצורכי בדיקות

חסרונות

למתחילים:

  • עלולה ליצור מספר רב של מחלקות
  • עלולה להיות מורכבת מדי עבור מקרים פשוטים
  • מקשה על הוספת חלקים חדשים לאובייקט

למתקדמים:

  • עלולה להקשות על הרחבת הממשק
  • עלולה ליצור תלות בין המחלקות
  • עלולה להקשות על הבנת הקוד
  • עלולה ליצור קוד מיותר במקרים פשוטים

מלכודות נפוצות

למתחילים:

  • יצירת יותר מדי מחלקות קונקרטיות
  • שימוש בתבנית כאשר לא נדרשת
  • אי-הקפדה על סדר הבנייה

למתקדמים:

  • אי-שימוש בממשקים מופשטים
  • אי-הפרדה בין הלוגיקה של בניית האובייקט לייצוג שלו
  • אי-שימוש ב-Director

השוואה לתבניות אחרות

למתחילים:

תבנית הבדל
Factory Method מתמקדת ביצירת אובייקט בודד
Abstract Factory מתמקדת ביצירת משפחות של מוצרים

למתקדמים:

תבנית הבדל
Factory Method מתמקדת ביצירת אובייקט בודד, בעוד Builder מתמקדת בבניית אובייקט מורכב
Abstract Factory מתמקדת ביצירת משפחות של מוצרים, בעוד Builder מתמקדת בבניית אובייקט מורכב
Prototype מתמקדת ביצירת אובייקטים על ידי שיבוט, בעוד Builder מתמקדת בבניית אובייקטים חדשים
Singleton מתמקדת ביצירת מופע יחיד של מחלקה, בעוד Builder מתמקדת בבניית אובייקטים חדשים

תרגול

למתחילים:

נסה ליישם את תבנית Builder במקרים הבאים:

  1. מערכת ניהול מסמכים עם תמיכה בפורמטים שונים
  2. מערכת ניהול קבצים עם תמיכה בסוגי קבצים שונים
  3. מערכת עיבוד תמונות עם תמיכה בפורמטים שונים

למתקדמים:

נסה ליישם את תבנית Builder במקרים הבאים:

  1. מערכת משחק עם דמויות שונות
  2. מערכת דיווח עם פורמטים שונים
  3. מערכת תקשורת עם פרוטוקולים שונים
  4. מערכת עיבוד תמונות עם פורמטים שונים

טיפים ליישום

למתחילים:

  • השתמש בממשקים מופשטים להגדרת המחלקות והאובייקטים
  • ודא שכל מחלקה יורשת מיישמת את הממשק המוגדר
  • הוסף בדיקות תקינות לאובייקטים שיוצרו

למתקדמים:

  • השתמש ב-Singleton עבור המחלקות הקונקרטיות אם נדרש
  • שקול שימוש ב-Registry או Configuration עבור בחירת המחלקה המתאימה
  • השתמש ב-Dependency Injection עבור הזרקת המחלקה
  • שקול שימוש ב-Abstract Factory בתוך Builder
  • הוסף תיעוד מפורט לממשקים והמימושים

מקרי שימוש נפוצים

למתחילים:

  • מערכות ניהול מסמכים
  • מערכות ניהול קבצים
  • מערכות עיבוד תמונות

למתקדמים:

  • מערכות משחק
  • מערכות דיווח
  • מערכות תקשורת
  • מערכות עיבוד תמונות
  • מערכות אחסון קבצים
