Strategy Pattern

חזרה לדף הבית

מטרת התבנית

תבנית Strategy מאפשרת להגדיר משפחה של אלגוריתמים, להפוך אותם להחלפים, ולהשתמש בהם באופן דינמי.

מתי להשתמש?

  • כאשר יש מספר אלגוריתמים שונים לביצוע אותה משימה
  • כאשר רוצים לאפשר החלפה דינמית של אלגוריתמים
  • כאשר רוצים להפריד בין האלגוריתם לבין הקוד שמשתמש בו
  • כאשר רוצים להימנע משימוש בתנאים מורכבים

דוגמה בסיסית 

from abc import ABC, abstractmethod

class Strategy(ABC):
    @abstractmethod
    def execute(self, data) -> None:
        pass

class ConcreteStrategyA(Strategy):
    def execute(self, data) -> None:
        print("ConcreteStrategyA: טיפול בנתונים")
        # טיפול בנתונים לפי אסטרטגיה A

class ConcreteStrategyB(Strategy):
    def execute(self, data) -> None:
        print("ConcreteStrategyB: טיפול בנתונים")
        # טיפול בנתונים לפי אסטרטגיה B

class Context:
    def __init__(self, strategy: Strategy) -> None:
        self._strategy = strategy

    @property
    def strategy(self) -> Strategy:
        return self._strategy

    @strategy.setter
    def strategy(self, strategy: Strategy) -> None:
        self._strategy = strategy

    def execute_strategy(self, data) -> None:
        self._strategy.execute(data)

# שימוש
if __name__ == "__main__":
    context = Context(ConcreteStrategyA())
    context.execute_strategy("נתונים")
    
    context.strategy = ConcreteStrategyB()
    context.execute_strategy("נתונים")

יתרונות

  • מאפשרת החלפה דינמית של אלגוריתמים
  • מפחיתה את הצורך בתנאים מורכבים
  • מאפשרת הוספה קלה של אלגוריתמים חדשים
  • מאפשרת הפרדה בין האלגוריתם לבין הקוד שמשתמש בו

חסרונות

  • עלולה ליצור מספר רב של מחלקות
  • עלולה להקשות על הבנת הקוד
  • עלולה לגרום לביצועים נמוכים
  • עלולה ליצור קוד קשה לתחזוקה
