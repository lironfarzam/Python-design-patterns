Template Method Pattern

חזרה לדף הבית

מטרת התבנית

תבנית Template Method מגדירה את השלד של אלגוריתם במחלקת בסיס ומאפשרת למחלקות יורשות להחליף חלקים מהאלגוריתם מבלי לשנות את המבנה הכללי.

מתי להשתמש?

  • כאשר יש מספר אלגוריתמים דומים עם הבדלים קטנים
  • כאשר רוצים למנוע כפילות קוד
  • כאשר רוצים לאפשר הרחבה של אלגוריתמים קיימים
  • כאשר רוצים לשלוט על נקודות הרחבה באלגוריתם

דוגמה בסיסית 

from abc import ABC, abstractmethod

class AbstractClass(ABC):
    def template_method(self) -> None:
        self.base_operation1()
        self.required_operations1()
        self.base_operation2()
        self.hook1()
        self.required_operations2()
        self.base_operation3()
        self.hook2()

    def base_operation1(self) -> None:
        print("AbstractClass: ביצוע פעולה בסיסית 1")

    def base_operation2(self) -> None:
        print("AbstractClass: ביצוע פעולה בסיסית 2")

    def base_operation3(self) -> None:
        print("AbstractClass: ביצוע פעולה בסיסית 3")

    @abstractmethod
    def required_operations1(self) -> None:
        pass

    @abstractmethod
    def required_operations2(self) -> None:
        pass

    def hook1(self) -> None:
        pass

    def hook2(self) -> None:
        pass

class ConcreteClass1(AbstractClass):
    def required_operations1(self) -> None:
        print("ConcreteClass1: ביצוע פעולה נדרשת 1")

    def required_operations2(self) -> None:
        print("ConcreteClass1: ביצוע פעולה נדרשת 2")

class ConcreteClass2(AbstractClass):
    def required_operations1(self) -> None:
        print("ConcreteClass2: ביצוע פעולה נדרשת 1")

    def required_operations2(self) -> None:
        print("ConcreteClass2: ביצוע פעולה נדרשת 2")

    def hook1(self) -> None:
        print("ConcreteClass2: ביצוע hook 1")

# שימוש
if __name__ == "__main__":
    concrete1 = ConcreteClass1()
    concrete1.template_method()

    print("\n")

    concrete2 = ConcreteClass2()
    concrete2.template_method()

יתרונות

  • מאפשרת שימוש חוזר בקוד
  • מאפשרת הרחבה קלה של אלגוריתמים
  • מאפשרת שליטה על נקודות הרחבה
  • מפחיתה כפילות קוד

חסרונות

  • עלולה ליצור היררכיה מורכבת
  • עלולה להגביל את הגמישות
  • עלולה להקשות על הבנת הקוד
  • עלולה ליצור תלות חזקה בין המחלקות

השוואה לתבניות אחרות

תבנית הבדל
Strategy משתמשת בהרכבה, בעוד Template Method משתמשת בירושה
Factory Method מתמקדת ביצירת אובייקטים, בעוד Template Method מתמקדת באלגוריתמים

תרגול

  2. מערכת הכנת משקאות
  3. מערכת בניית דוחות
