Prototype Pattern

חזרה לדף הבית

מטרת התבנית

תבנית Prototype מאפשרת יצירת אובייקטים חדשים על ידי העתקת אובייקט קיים, במקום ליצור אותם מאפס.

מתי להשתמש?

  • כאשר רוצים להימנע מיצירת אובייקטים חדשים מאפס
  • כאשר רוצים ליצור אובייקטים דומים לאובייקטים קיימים
  • כאשר רוצים להפחית את מספר המחלקות במערכת
  • כאשר רוצים להימנע משימוש ב-inheritance

דוגמה בסיסית 

from abc import ABC, abstractmethod
from copy import deepcopy

class Prototype(ABC):
    @abstractmethod
    def clone(self):
        pass

class ConcretePrototype1(Prototype):
    def __init__(self, value: str):
        self.value = value

    def clone(self):
        return deepcopy(self)

class ConcretePrototype2(Prototype):
    def __init__(self, value: str):
        self.value = value

    def clone(self):
        return deepcopy(self)

# שימוש
def client_code(prototype: Prototype) -> None:
    clone = prototype.clone()
    print(f"Original: {prototype.value}")
    print(f"Clone: {clone.value}")

if __name__ == "__main__":
    prototype1 = ConcretePrototype1("ערך 1")
    client_code(prototype1)

    prototype2 = ConcretePrototype2("ערך 2")
    client_code(prototype2)

יתרונות

  • מאפשרת יצירת אובייקטים חדשים ללא תלות במחלקות
  • מאפשרת יצירת אובייקטים חדשים ללא תלות בקונסטרוקטור
  • מאפשרת יצירת אובייקטים חדשים ללא תלות בממשק
  • מאפשרת יצירת אובייקטים חדשים ללא תלות במימוש

חסרונות

  • עלולה ליצור קוד מורכב
  • עלולה ליצור קוד קשה לתחזוקה
  • עלולה ליצור קוד קשה להבנה
  • עלולה ליצור קוד קשה לבדיקה

השוואה לתבניות אחרות

תבנית הבדל
Factory Method מתמקדת ביצירת אובייקטים חדשים, בעוד Prototype מתמקדת בהעתקת אובייקטים קיימים
Singleton מתמקדת במופע יחיד, בעוד Prototype מתמקדת בהעתקת אובייקטים

תרגול

נסה ליישם את תבנית Prototype במקרים הבאים:

  1. מערכת ניהול מסמכים עם תבניות
  2. מערכת ניהול משתמשים עם פרופילים
  3. מערכת ניהול מוצרים עם קטגוריות
