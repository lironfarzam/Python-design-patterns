Iterator Pattern

חזרה לדף הבית

מטרת התבנית

תבנית Iterator מאפשרת גישה רציפה לאלמנטים של מבנה נתונים מבלי לחשוף את המבנה הפנימי שלו.

מתי להשתמש?

  • כאשר רוצים לגשת לאלמנטים של מבנה נתונים מבלי לחשוף את המבנה הפנימי
  • כאשר רוצים לתמוך במספר דרכים שונות לעבור על מבנה נתונים
  • כאשר רוצים לספק ממשק אחיד למבני נתונים שונים

דוגמה בסיסית 

from abc import ABC, abstractmethod
from typing import Any, List, Optional

# ממשק משותף לאיטרטור
class Iterator(ABC):
    @abstractmethod
    def next(self) -> Optional[Any]:
        pass

    @abstractmethod
    def has_next(self) -> bool:
        pass

# ממשק משותף למבנה נתונים
class Aggregate(ABC):
    @abstractmethod
    def create_iterator(self) -> Iterator:
        pass

# מימוש קונקרטי של איטרטור
class ConcreteIterator(Iterator):
    def __init__(self, items: List[Any]):
        self._items = items
        self._index = 0

    def next(self) -> Optional[Any]:
        if self.has_next():
            item = self._items[self._index]
            self._index += 1
            return item
        return None

    def has_next(self) -> bool:
        return self._index < len(self._items)

# מימוש קונקרטי של מבנה נתונים
class ConcreteAggregate(Aggregate):
    def __init__(self):
        self._items: List[Any] = []

    def add_item(self, item: Any) -> None:
        self._items.append(item)

    def create_iterator(self) -> Iterator:
        return ConcreteIterator(self._items)

# שימוש
def client_code() -> None:
    aggregate = ConcreteAggregate()
    aggregate.add_item("א")
    aggregate.add_item("ב")
    aggregate.add_item("ג")

    iterator = aggregate.create_iterator()
    while iterator.has_next():
        print(iterator.next())

יתרונות

  • מאפשר גישה לאלמנטים מבלי לחשוף את המבנה הפנימי
  • מאפשר מספר דרכים שונות לעבור על מבנה נתונים
  • מאפשר שינוי המבנה הפנימי ללא השפעה על הקוד החיצוני
  • מאפשר לעצור את המעבר בכל רגע

חסרונות

  • עלול להגדיל את המורכבות של הקוד
  • עלול להשפיע על הביצועים
  • לא מתאים למבני נתונים פשוטים
