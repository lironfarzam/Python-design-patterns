Facade Pattern

חזרה לדף הבית

מהי תבנית Facade?

תבנית Facade היא תבנית עיצוב מבנית המספקת ממשק פשוט ונוח לשימוש עבור מערכת מורכבת. התבנית משמשת כאשר יש צורך להסתיר את המורכבות של מערכת ולהציג ממשק פשוט למשתמש.

הסבר למתחילים:

דמיינו שאתם רוצים להזמין פיצה. אתם לא צריכים לדעת איך הפיצה מיוצרת, איך הבצק מוכן, או איך הגבינה נחתכת. אתם פשוט מתקשרים למסעדה ומבקשים פיצה. המסעדה משמשת כ-Facade - היא מסתירה את המורכבות של תהליך הכנת הפיצה ומספקת לכם ממשק פשוט.

הסבר למתקדמים:

התבנית מיישמת את עקרון ה-Single Responsibility Principle על ידי הפרדת הממשק מהמימוש. זה מאפשר שינוי המימוש מבלי להשפיע על הממשק, ומאפשר שימוש חוזר בקוד קיים.

מתי להשתמש בתבנית?

למתחילים:

  • כאשר אתם צריכים להסתיר את המורכבות של מערכת
  • כאשר אתם צריכים להציג ממשק פשוט למשתמש
  • כאשר אתם צריכים להפריד בין הממשק למימוש

למתקדמים:

  • כאשר יש צורך להסתיר את המורכבות של מערכת
  • כאשר יש צורך להציג ממשק פשוט למשתמש
  • כאשר יש צורך להפריד בין הממשק למימוש
  • כאשר יש צורך לשמור על עקרון ה-Single Responsibility Principle
  • כאשר יש צורך לאפשר שימוש חוזר בקוד קיים
  • כאשר יש צורך לאפשר שינוי המימוש מבלי להשפיע על הממשק

מבנה התבנית

הסבר בסיסי:

התבנית מורכבת משלושה חלקים עיקריים:

  • Facade - הממשק הפשוט שמספק גישה למערכת המורכבת
  • Subsystem - המערכת המורכבת שמספקת את הפונקציונליות
  • Client - המשתמש שמשתמש ב-Facade

הסבר מתקדם:

התבנית מורכבת מהמרכיבים הבאים:

  • Facade - הממשק הפשוט שמספק גישה למערכת המורכבת
  • Subsystem - המערכת המורכבת שמספקת את הפונקציונליות
  • Client - המשתמש שמשתמש ב-Facade
  • Additional Facade - ממשקים נוספים שמספקים גישה לחלקים ספציפיים של המערכת

היחסים בין המרכיבים:

  • Facade מגדיר את הממשק הפשוט
  • Subsystem מספק את הפונקציונליות
  • Client משתמש ב-Facade
  • Additional Facade מספק גישה לחלקים ספציפיים

דוגמה בסיסית

הסבר למתחילים:

בדוגמה הבאה, אנחנו יוצרים מערכת לניהול מסעדה. יש לנו מערכת מורכבת שמטפלת בהזמנות, הכנת אוכל, וקבלת תשלום. ה-Facade מספק ממשק פשוט למשתמש.

הסבר למתקדמים:

הדוגמה מדגימה את השימוש ב-Facade Pattern להסתרת המורכבות של מערכת. הממשק הפשוט מגדיר את הפעולות הבסיסיות, והמימושים הספציפיים מספקים את הפונקציונליות.

class OrderSystem:
    def place_order(self, order):
        print(f"מקבל הזמנה: {order}")
    
    def cancel_order(self, order):
        print(f"מבטל הזמנה: {order}")

class KitchenSystem:
    def prepare_food(self, order):
        print(f"מכין אוכל: {order}")
    
    def package_food(self, order):
        print(f"מארז אוכל: {order}")

class PaymentSystem:
    def process_payment(self, amount):
        print(f"מעבד תשלום: {amount} ש\"ח")
    
    def refund_payment(self, amount):
        print(f"מחזיר תשלום: {amount} ש\"ח")

class RestaurantFacade:
    def __init__(self):
        self._order_system = OrderSystem()
        self._kitchen_system = KitchenSystem()
        self._payment_system = PaymentSystem()
    
    def order_food(self, order, amount):
        self._order_system.place_order(order)
        self._kitchen_system.prepare_food(order)
        self._kitchen_system.package_food(order)
        self._payment_system.process_payment(amount)
    
    def cancel_order(self, order, amount):
        self._order_system.cancel_order(order)
        self._payment_system.refund_payment(amount)

# שימוש
if __name__ == "__main__":
    restaurant = RestaurantFacade()
    
    # הזמנת אוכל
    restaurant.order_food("פיצה", 50)
    
    # ביטול הזמנה
    restaurant.cancel_order("פיצה", 50)

דוגמה מעשית - מערכת ניהול חשבונות

הסבר למתחילים:

בדוגמה זו, אנחנו יוצרים מערכת לניהול חשבונות. יש לנו מערכת מורכבת שמטפלת בחשבונות, תשלומים, ודוחות. ה-Facade מספק ממשק פשוט למשתמש.

הסבר למתקדמים:

הדוגמה מדגימה את השימוש ב-Facade Pattern להסתרת המורכבות של מערכת. הממשק הפשוט מגדיר את הפעולות הבסיסיות, והמימושים הספציפיים מספקים את הפונקציונליות.

class AccountSystem:
    def create_account(self, customer):
        print(f"יוצר חשבון עבור: {customer}")
    
    def close_account(self, customer):
        print(f"סוגר חשבון עבור: {customer}")

class PaymentSystem:
    def process_payment(self, customer, amount):
        print(f"מעבד תשלום עבור {customer}: {amount} ש\"ח")
    
    def refund_payment(self, customer, amount):
        print(f"מחזיר תשלום עבור {customer}: {amount} ש\"ח")

class ReportSystem:
    def generate_report(self, customer):
        print(f"מייצר דוח עבור: {customer}")
    
    def send_report(self, customer):
        print(f"שולח דוח ל: {customer}")

class BankingFacade:
    def __init__(self):
        self._account_system = AccountSystem()
        self._payment_system = PaymentSystem()
        self._report_system = ReportSystem()
    
    def open_account(self, customer):
        self._account_system.create_account(customer)
        self._report_system.generate_report(customer)
    
    def close_account(self, customer):
        self._account_system.close_account(customer)
        self._report_system.generate_report(customer)
    
    def make_payment(self, customer, amount):
        self._payment_system.process_payment(customer, amount)
        self._report_system.generate_report(customer)
        self._report_system.send_report(customer)
    
    def refund_payment(self, customer, amount):
        self._payment_system.refund_payment(customer, amount)
        self._report_system.generate_report(customer)
        self._report_system.send_report(customer)

# שימוש
if __name__ == "__main__":
    bank = BankingFacade()
    
    # פתיחת חשבון
    bank.open_account("ישראל ישראלי")
    
    # ביצוע תשלום
    bank.make_payment("ישראל ישראלי", 1000)
    
    # החזרת תשלום
    bank.refund_payment("ישראל ישראלי", 500)
    
    # סגירת חשבון
    bank.close_account("ישראל ישראלי")

השוואה לתבניות אחרות

למתחילים:

תבנית הבדל
Adapter מתמקדת בהתאמת ממשקים, בעוד Facade מתמקדת בהסתרת מורכבות
Mediator מתמקדת בתיווך בין אובייקטים, בעוד Facade מתמקדת בהסתרת מורכבות

למתקדמים:

תבנית הבדל
Adapter מתמקדת בהתאמת ממשקים, בעוד Facade מתמקדת בהסתרת מורכבות
Mediator מתמקדת בתיווך בין אובייקטים, בעוד Facade מתמקדת בהסתרת מורכבות
Proxy מתמקדת בשליטה בגישה, בעוד Facade מתמקדת בהסתרת מורכבות
Abstract Factory מתמקדת ביצירת משפחות של אובייקטים, בעוד Facade מתמקדת בהסתרת מורכבות
