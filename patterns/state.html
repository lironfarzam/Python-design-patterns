State Pattern

חזרה לדף הבית

מטרת התבנית

תבנית State מאפשרת לאובייקט לשנות את התנהגותו כאשר מצבו הפנימי משתנה.

מתי להשתמש?

  • כאשר אובייקט משנה את התנהגותו בהתאם למצבו
  • כאשר יש מספר רב של תנאים שמשפיעים על ההתנהגות
  • כאשר רוצים להימנע משימוש בתנאים מורכבים
  • כאשר רוצים להפריד בין מצבים שונים

דוגמה בסיסית 

from abc import ABC, abstractmethod

class State(ABC):
    @abstractmethod
    def handle(self, context) -> None:
        pass

class ConcreteStateA(State):
    def handle(self, context) -> None:
        print("ConcreteStateA: טיפול")
        context.state = ConcreteStateB()

class ConcreteStateB(State):
    def handle(self, context) -> None:
        print("ConcreteStateB: טיפול")
        context.state = ConcreteStateA()

class Context:
    def __init__(self) -> None:
        self._state = ConcreteStateA()

    @property
    def state(self) -> State:
        return self._state

    @state.setter
    def state(self, state: State) -> None:
        self._state = state

    def request(self) -> None:
        self._state.handle(this)

# שימוש
if __name__ == "__main__":
    context = Context()
    context.request()
    context.request()

יתרונות

  • מאפשרת הפרדה בין מצבים שונים
  • מפחיתה את הצורך בתנאים מורכבים
  • מאפשרת הוספה קלה של מצבים חדשים
  • מאפשרת שינוי התנהגות בזמן ריצה

חסרונות

  • עלולה ליצור קוד מורכב
  • עלולה ליצור קוד קשה לתחזוקה
  • עלולה ליצור קוד קשה להבנה
  • עלולה ליצור קוד קשה לבדיקה
