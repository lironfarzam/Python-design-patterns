Template Method Pattern

חזרה לדף הבית

מהי תבנית Template Method?

תבנית Template Method היא תבנית עיצוב התנהגותית המגדירה את השלד של אלגוריתם במחלקה בסיסית, ומאפשרת למחלקות יורשות לשנות חלקים מהאלגוריתם מבלי לשנות את המבנה הכללי שלו.

הסבר למתחילים:

דמיינו שיש לכם מתכון להכנת עוגה. המתכון מגדיר את השלבים הבסיסיים: ערבוב החומרים, הכנסה לתנור, ובדיקת מוכנות. כל שלב יכול להיות מותאם לפי סוג העוגה. תבנית Template Method עובדת באותו אופן - היא מגדירה את השלבים הבסיסיים ומאפשרת התאמה של כל שלב.

הסבר למתקדמים:

התבנית מיישמת את עקרון ה-Hollywood Principle ("אל תתקשר אלינו, אנחנו נתקשר אליך") על ידי הפרדה בין האלגוריתם הבסיסי לבין המימושים הספציפיים. המחלקה הבסיסית מגדירה את השלבים, והמחלקות היורשות מספקות את המימושים הספציפיים.

מתי להשתמש בתבנית?

למתחילים:

  • כאשר יש צורך להגדיר את השלד של אלגוריתם
  • כאשר יש צורך לאפשר התאמה של חלקים מהאלגוריתם
  • כאשר יש צורך למנוע כפילות בקוד

למתקדמים:

  • כאשר יש צורך להגדיר את השלד של אלגוריתם
  • כאשר יש צורך לאפשר התאמה של חלקים מהאלגוריתם
  • כאשר יש צורך למנוע כפילות בקוד
  • כאשר יש צורך לשמור על עקרון ה-Open/Closed Principle
  • כאשר יש צורך להפחית את הצימוד בין המחלקות
  • כאשר יש צורך לאפשר הרחבה של המערכת עם אלגוריתמים חדשים

מבנה התבנית

הסבר בסיסי:

התבנית מורכבת משני חלקים עיקריים:

  • AbstractClass - המחלקה הבסיסית שמגדירה את השלד של האלגוריתם
  • ConcreteClass - המחלקה היורשת שמספקת את המימושים הספציפיים

הסבר מתקדם:

התבנית מורכבת מהמרכיבים הבאים:

  • AbstractClass - המחלקה הבסיסית שמגדירה את השלד של האלגוריתם
  • ConcreteClass - המחלקה היורשת שמספקת את המימושים הספציפיים
  • Template Method - המתודה שמגדירה את השלד של האלגוריתם
  • Hook Methods - המתודות שמאפשרות התאמה של חלקים מהאלגוריתם

היחסים בין המרכיבים:

  • AbstractClass מגדיר את השלד של האלגוריתם
  • ConcreteClass מיישם את המתודות הספציפיות
  • Template Method מגדיר את השלבים
  • Hook Methods מאפשרים התאמה

דוגמה בסיסית

הסבר למתחילים:

בדוגמה הבאה, אנחנו יוצרים מערכת להכנת משקאות. המערכת מגדירה את השלבים הבסיסיים להכנת משקאות, ומאפשרת התאמה של כל שלב לפי סוג המשקה.

הסבר למתקדמים:

הדוגמה מדגימה את השימוש ב-Template Method Pattern להפרדה בין האלגוריתם הבסיסי לבין המימושים הספציפיים. המחלקה הבסיסית מגדירה את השלבים, והמחלקות היורשות מספקות את המימושים הספציפיים.

from abc import ABC, abstractmethod

class Beverage(ABC):
    def prepare(self) -> None:
        self.boil_water()
        self.brew()
        self.pour_in_cup()
        if self.customer_wants_condiments():
            self.add_condiments()
    
    def boil_water(self) -> None:
        print("מרתיחים מים")
    
    def pour_in_cup(self) -> None:
        print("יוצקים לכוס")
    
    def customer_wants_condiments(self) -> bool:
        return True
    
    @abstractmethod
    def brew(self) -> None:
        pass
    
    @abstractmethod
    def add_condiments(self) -> None:
        pass

class Tea(Beverage):
    def brew(self) -> None:
        print("מכינים תה")
    
    def add_condiments(self) -> None:
        print("מוסיפים לימון")

class Coffee(Beverage):
    def brew(self) -> None:
        print("מכינים קפה")
    
    def add_condiments(self) -> None:
        print("מוסיפים סוכר וחלב")
    
    def customer_wants_condiments(self) -> bool:
        answer = input("האם תרצה סוכר וחלב? (כן/לא): ")
        return answer.lower() == "כן"

# שימוש
if __name__ == "__main__":
    # הכנת תה
    tea = Tea()
    tea.prepare()
    
    # הכנת קפה
    coffee = Coffee()
    coffee.prepare()

דוגמה מעשית - מערכת ניהול משימות

הסבר למתחילים:

בדוגמה זו, אנחנו יוצרים מערכת ניהול משימות. המערכת מגדירה את השלבים הבסיסיים לניהול משימה, ומאפשרת התאמה של כל שלב לפי סוג המשימה.

הסבר למתקדמים:

הדוגמה מדגימה את השימוש ב-Template Method Pattern להפרדה בין האלגוריתם הבסיסי לבין המימושים הספציפיים. המחלקה הבסיסית מגדירה את השלבים, והמחלקות היורשות מספקות את המימושים הספציפיים.

from abc import ABC, abstractmethod
from datetime import datetime
from typing import List, Optional

class Task(ABC):
    def __init__(self, title: str):
        self._title = title
        self._created_at = datetime.now()
        self._completed_at: Optional[datetime] = None
        self._subtasks: List[Task] = []
    
    def execute(self) -> None:
        self._validate()
        self._prepare()
        self._process()
        self._cleanup()
        self._complete()
    
    def _validate(self) -> None:
        print(f"בודקים את המשימה: {self._title}")
    
    def _prepare(self) -> None:
        print(f"מכינים את המשימה: {self._title}")
    
    def _process(self) -> None:
        print(f"מבצעים את המשימה: {self._title}")
        self._execute_subtasks()
    
    def _cleanup(self) -> None:
        print(f"מנקים אחרי המשימה: {self._title}")
    
    def _complete(self) -> None:
        self._completed_at = datetime.now()
        print(f"המשימה הושלמה: {self._title}")
    
    def _execute_subtasks(self) -> None:
        for subtask in self._subtasks:
            subtask.execute()
    
    def add_subtask(self, subtask: 'Task') -> None:
        self._subtasks.append(subtask)
    
    @abstractmethod
    def _validate(self) -> None:
        pass
    
    @abstractmethod
    def _prepare(self) -> None:
        pass
    
    @abstractmethod
    def _process(self) -> None:
        pass
    
    @abstractmethod
    def _cleanup(self) -> None:
        pass

class DevelopmentTask(Task):
    def _validate(self) -> None:
        print(f"בודקים את הדרישות של המשימה: {self._title}")
    
    def _prepare(self) -> None:
        print(f"מכינים את הסביבה לפיתוח: {self._title}")
    
    def _process(self) -> None:
        print(f"מפתחים את הקוד: {self._title}")
        self._execute_subtasks()
    
    def _cleanup(self) -> None:
        print(f"מנקים את הקוד: {self._title}")

class TestingTask(Task):
    def _validate(self) -> None:
        print(f"בודקים את הדרישות של הבדיקות: {self._title}")
    
    def _prepare(self) -> None:
        print(f"מכינים את הסביבה לבדיקות: {self._title}")
    
    def _process(self) -> None:
        print(f"מבצעים את הבדיקות: {self._title}")
        self._execute_subtasks()
    
    def _cleanup(self) -> None:
        print(f"מנקים את תוצאות הבדיקות: {self._title}")

# שימוש
if __name__ == "__main__":
    # יצירת משימת פיתוח
    dev_task = DevelopmentTask("פיתוח מודול חדש")
    
    # יצירת משימת בדיקות
    test_task = TestingTask("בדיקת המודול")
    
    # הוספת משימת בדיקות כתת-משימה
    dev_task.add_subtask(test_task)
    
    # ביצוע המשימה
    dev_task.execute()
