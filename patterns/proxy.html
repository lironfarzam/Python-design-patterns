Proxy Pattern

חזרה לדף הבית

מטרת התבנית

תבנית Proxy מספקת נציג או תחליף לאובייקט אחר, ומאפשרת שליטה בגישה אליו.

מתי להשתמש?

  • כאשר רוצים לשלוט בגישה לאובייקט
  • כאשר רוצים להוסיף פונקציונליות לאובייקט
  • כאשר רוצים להפחית את העומס על האובייקט
  • כאשר רוצים להסתיר את המימוש של האובייקט

דוגמה בסיסית 

from abc import ABC, abstractmethod

class Subject(ABC):
    @abstractmethod
    def request(self) -> None:
        pass

class RealSubject(Subject):
    def request(self) -> None:
        print("RealSubject: טיפול בבקשה")

class Proxy(Subject):
    def __init__(self, real_subject: RealSubject):
        self._real_subject = real_subject

    def request(self) -> None:
        if self.check_access():
            self._real_subject.request()
            self.log_access()

    def check_access(self) -> bool:
        print("Proxy: בדיקת הרשאות")
        return True

    def log_access(self) -> None:
        print("Proxy: רישום גישה")

# שימוש
def client_code(subject: Subject) -> None:
    subject.request()

if __name__ == "__main__":
    real_subject = RealSubject()
    client_code(real_subject)

    proxy = Proxy(real_subject)
    client_code(proxy)

יתרונות

  • מאפשרת שליטה בגישה לאובייקט
  • מאפשרת הוספת פונקציונליות לאובייקט
  • מאפשרת הפחתת עומס על האובייקט
  • מאפשרת הסתרת מימוש של האובייקט

חסרונות

  • עלולה ליצור קוד מורכב
  • עלולה ליצור קוד קשה לתחזוקה
  • עלולה ליצור קוד קשה להבנה
  • עלולה ליצור קוד קשה לבדיקה

השוואה לתבניות אחרות

תבנית הבדל
Decorator מוסיפה תפקוד לאובייקט, בעוד Proxy מתמקדת בשליטה בגישה
Adapter מתאימה ממשקים, בעוד Proxy מתמקדת בשליטה בגישה

תרגול

נסה ליישם את תבנית Proxy במקרים הבאים:

  1. מערכת ניהול קבצים עם הרשאות
  2. מערכת תקשורת עם אבטחה
  3. מערכת ניהול בסיסי נתונים עם זיכרון מטמון
